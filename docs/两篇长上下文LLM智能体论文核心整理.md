# 两篇长上下文LLM智能体论文核心整理

两篇均聚焦**长时序多轮工具/搜索智能体**的**上下文管理+强化学习(RL)** 优化，分别提出**摘要式上下文管理器**与**滑动窗口式上下文管理器**，突破固定上下文窗口限制。

## 一、论文1：arXiv:2510.06727（SummarizerContextManager）

### 基础信息

- 标题：*Scaling LLM Multi-turn RL with End-to-end Summarization-based Context Management*

- 团队：ByteDance Seed、斯坦福大学、卡内基梅隆大学

- 核心组件：**SummarizerContextManager（摘要式上下文管理器）**

- 核心算法：**SUPO（摘要增强策略优化）**

### 解决核心问题

长时序多轮工具调用RL训练的三大瓶颈：

1. 长上下文导致指令遵循能力退化

2. 轨迹Rollout计算成本过高

3. 固定上下文长度锁死任务复杂度上限

### 核心技术方案

1. **摘要增强MDP建模**

定期将工具调用历史压缩为任务相关摘要，上下文重置为「初始Prompt+历史摘要」，保持紧凑且信息完整。

1. **端到端策略梯度推导**

长轨迹拆分为多个摘要子轨迹，兼容现有LLM RL基础设施，**联合优化工具调用与摘要策略**。

1. **SUPO算法关键设计**

    - 轨迹管理：拆分长轨迹适配现有RL框架

    - 分组相对优势估计：同轨迹共享优势值，稳定训练

    - 超长轨迹掩码：过滤无效梯度，避免摘要策略崩溃

    - 上下文精控：丢弃末尾动作-观测对，严格约束长度

### 实验结果

- 测试环境：CodeGym（函数调用）、BrowseComp-Plus（深度搜索）

- 核心效果：

    - CodeGym：4K工作上下文达到基线32K效果，准确率**+3.2%**

    - BrowseComp-Plus：64K工作上下文（有效192K），准确率**+14.0%**

- 测试时扩展：增加摘要轮数，性能最高再**+7.0%**

### 核心贡献

1. 提出**端到端摘要式上下文管理**，将摘要作为策略一部分联合优化

2. 推导摘要增强MDP的策略梯度，无缝适配现有RL体系

3. SUPO在长时序工具/搜索任务显著优于基线GRPO

## 二、论文2：arXiv:2510.08276（SlidingWindowContextManager）

### 基础信息

- 标题：*Beyond Turn Limits: Training Deep Search Agents with Dynamic Context Window*

- 团队：中科院软件所、阿里巴巴、中国科学院大学

- 核心组件：**SlidingWindowContextManager（动态滑动窗口上下文管理器）**

- 核心框架：**DeepMiner**

### 解决核心问题

1. 现有训练任务复杂度低，无法激发深度推理能力

2. 上下文爆炸：32K仅支持10–15轮交互；摘要方案丢失细粒度信息、增加开销

3. 训练-推理上下文状态不一致

### 核心技术方案

1. **高难度任务反向构建**

从真实网页生成需跨4源推理的复杂QA，模糊化提升难度，严格过滤保证可验证性。

1. **动态滑动窗口上下文管理**

    - 仅压缩**早期工具返回**为占位符，**完整保留助手推理轨迹**

    - 窗口大小W/步长S触发滑动，替换过期工具响应，不损失推理逻辑

2. **训练-推理一致性**

长轨迹拆分为多训练序列，掩码保证仅新响应更新梯度，避免优化冲突

1. **两阶段训练**

    - SFT冷启动：学习工具调用与长时序推理

    - GRPO RL优化：轨迹级优势共享到子序列，二元奖励简化设计

### 实验结果

- 基座：Qwen3-32B → DeepMiner-32B

- 测试基准：BrowseComp-en/zh、XBench-DeepSearch、GAIA

- 核心效果：

    - BrowseComp-en：**33.5%**准确率，超开源SOTA近20个点

    - 32K上下文支持**近100轮**交互，效率远超原生/摘要方案

- 数据效率：自建数据集优于HotpotQA，SFT即超多数开源智能体

### 核心贡献

1. 反向构建**高难度可验证深度搜索任务**，填补训练数据短板

2. **无外部摘要的动态滑动窗口**，保留推理轨迹、高效利用上下文

3. DeepMiner大幅刷新深度搜索开源SOTA
